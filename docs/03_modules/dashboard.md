# Dashboard Module

The dashboard is the primary user interface for SRCI.

---

## Purpose

To expose SRCI intelligence through structured, visual workflows rather than chat-based interaction.

---

## Core Screens

### System Overview
- Service dependency graph
- Recent changes
- Recent incidents

### Change Impact View
- Change summary
- Impacted components
- Risk score and explanation

### Incident Investigation View
- Incident timeline
- Causal graph
- Root cause hypotheses

### History & Learning
- Past incidents
- Linked changes
- Recurrent failure patterns

---

## Design Principles

- Visual-first reasoning
- Minimal interaction friction
- Explainability and traceability
- No real-time monitoring
